# 🎓 เครื่องคำนวณเกรดเฉลี่ย (GPA Calculator)
เครื่องคำนวณเกรดเฉลี่ยรายเทอมแบบโต้ตอบได้แบบเรียลไทม์ พัฒนาด้วยเทคโนโลยีเว็บพื้นฐานมาตรฐานระดับสูง (HTML, CSS และ Vanilla JavaScript) ออกแบบมาในสไตล์ **Glassmorphism (กระจกฝ้าพรีเมียม)** ที่สวยงาม สะอาดตา และใช้งานง่ายบนทุกอุปกรณ์ (Fully Responsive)

## ✨ ฟีเจอร์ที่น่าสนใจ (Key Features)
1. **Dynamic Rows**: สามารถเพิ่มวิชาเรียนใหม่ได้อย่างไม่จำกัด หรือลบรายวิชาที่ไม่ต้องการออกได้อย่างอิสระผ่านปุ่มเพิ่มและลบ
2. **Auto-Calculation**: คำนวณเกรดเฉลี่ยสะสม (GPA) ให้อัตโนมัติทุกครั้งที่มีการเปลี่ยนค่าหน่วยกิตหรือเกรด โดยไม่ต้องกดปุ่มคำนวณ
3. **Responsive Design**: การ์ดอินพุตจะจัดรูปแบบใหม่ให้เข้ากับหน้าจอมือถือโดยอัตโนมัติ ทำให้ผู้ใช้บนโทรศัพท์กรอกข้อมูลได้อย่างไม่มีสะดุด
4. **Local State Persistence**: มีระบบบันทึกรายวิชาลงในบราวเซอร์ (LocalStorage) ทำให้แม้จะปิดแท็บหรือรีเฟรชหน้าเว็บ ข้อมูลที่เคยกรอกไว้ก็จะไม่หายไป
5. **XSS Protection**: มีระบบ Escape ข้อความดิบ ป้องกันการกรอกข้อมูลประสงค์ร้ายเพื่อรันสคริปต์ในระบบ

## 🗂️ โครงสร้างไฟล์ในโปรเจกต์
ตามหลักการ **Separation of Concerns** (การแบ่งแยกหน้าที่อย่างชัดเจนตามมาตรฐานวิชา):
* `index.html`: กำหนดโครงสร้างหน้าเว็บและหน้าตากล่องผลลัพธ์ (ไม่มี inline style และไม่มีการเรียก onclick ใดๆ)
* `style.css`: จัดสไตล์การแสดงผล ระบบสี Dark Mode เอฟเฟกต์เบลอขอบกระจก แอนิเมชัน และการออกแบบ responsive
* `script.js`: จัดการ Event Logic ทั้งหมด, คำนวณเกรดเฉลี่ยแบบเรียลไทม์ และเก็บข้อมูลลง LocalStorage

## 🧮 สูตรคำนวณเกรดเฉลี่ย (GPA Formula)
การหาเกรดเฉลี่ยใช้สูตรแบบถ่วงน้ำหนักตามหน่วยกิตวิชา:

$$GPA = \frac{\sum (\text{หน่วยกิตของวิชานั้น} \times \text{คะแนนประจำเกรดของวิชานั้น})}{\text{ผลรวมหน่วยกิตทั้งหมดของวิชาที่ระบุเกรด}}$$

**ค่าคะแนนประจำเกรด (Grade Point Mapping):**
* A = `4.0` | B+ = `3.5` | B = `3.0` | C+ = `2.5` | C = `2.0` | D+ = `1.5` | D = `1.0` | F = `0.0`
* วิชาที่ยังไม่เลือกเกรด จะไม่นำมารวมในผลรวมหน่วยกิตและผลรวมคะแนนสะสม

## 📖 โพยเตรียมตอบคำถามอาจารย์ (Oral Exam Guide)

### ❓ คำถาม 1: แยกแยะ Separation of Concerns อย่างไร?
> **แนวคำตอบ**: ผม/หนู ได้แยกโครงสร้างออกจากสไตล์และพฤติกรรมอย่างชัดเจนครับ/ค่ะ:
> * ใน `index.html` จะไม่มีการใส่แอตทริบิวต์ `style="..."` เลย ทุกอย่างถูกดึงไปจัดการใน `style.css`
> * ใน HTML จะไม่มี Event Handler อย่างเช่น `onclick="..."` หรือ `onchange="..."` เลย แต่ใน `script.js` จะใช้คำสั่ง `addEventListener` ในการดักฟังเหตุการณ์การกระทำของผู้ใช้หลังจากที่ DOM โหลดเสร็จเรียบร้อย

### ❓ คำถาม 2: ในไฟล์ `script.js` ฟังก์ชัน `calculateGPA()` ทำงานอย่างไร?
> **แนวคำตอบ**: ฟังก์ชันนี้ทำงานดังนี้ครับ/ค่ะ:
> 1. ใช้ `querySelectorAll('.course-row')` ดึงทุกแถววิชาที่มีอยู่บนหน้าจอขึ้นมา
> 2. วนลูปตรวจสอบทีละแถวเพื่อเช็คค่าหน่วยกิตและเกรดที่เลือก โดยแปลงหน่วยกิตเป็นทศนิยมด้วย `parseFloat`
> 3. ถ้าเกรดที่เลือกไม่ใช่ค่าว่าง จะแปลงเกรดอักษรเป็นตัวเลขโดยแมปผ่านออบเจกต์ `GRADE_POINTS`
> 4. คูณหน่วยกิตกับเกรดสะสมไปเก็บไว้ในตัวแปร `totalPoints` และบวกหน่วยกิตสะสมไปที่ `totalCredits`
> 5. นำเอา `totalPoints` มาหารกับ `totalCredits` และครอบด้วยคำสั่ง `.toFixed(2)` เพื่อแปลงค่าเกรดเป็นทศนิยม 2 ตำแหน่ง นำไปแสดงผล

### ❓ คำถาม 3: ทำไมเว็บนี้จึงรันบน GitHub Pages ได้โดยไม่ต้องมีเซิร์ฟเวอร์?
> **แนวคำตอบ**: เพราะ GitHub Pages เป็นระบบเว็บโฮสติ้งแบบ **Static Web Hosting** ครับ/ค่ะ ซึ่งให้บริการไฟล์ที่ประกอบด้วย HTML, CSS และ JavaScript สำเร็จรูปที่ประมวลผลฝั่งไคลเอนต์ (Client-Side) ทั้งหมด ไม่จำเป็นต้องมี Node.js, PHP หรือฐานข้อมูลฝั่งเซิร์ฟเวอร์ (Server-Side) บราวเซอร์ของผู้ใช้งานจะเป็นคนรันโค้ดทั้งหมดนี้ด้วยตัวเอง

### ❓ คำถาม 4: ถ้าต้องการเพิ่มปุ่มสำหรับ "คำนวณแบบจำลอง" หรือแก้สูตร ต้องแก้ไฟล์ใดบ้าง?
> **แนวคำตอบ**: 
> * หากต้องการเพิ่ม UI/โครงสร้างปุ่ม ต้องเพิ่มโค้ดปุ่มลงใน `index.html`
> * หากต้องการตกแต่งความสวยงามของปุ่ม ต้องแก้ใน `style.css`
> * หากต้องการกำหนดลักษณะการทำงานเมื่อปุ่มถูกกด ต้องมาผูก event และเขียนตรรกะใหม่ใน `script.js`

### ❓ คำถาม 5: หากแก้ไขโค้ดที่เครื่องตนเองแล้ว ทำไมบนเว็บจริงยังเป็นรุ่นเก่า?
> **แนวคำตอบ**: อาจเกิดได้จาก 2 สาเหตุหลักครับ/ค่ะ:
> 1. **ยังไม่ได้ Commit และ Push โค้ด**: ต้องทำการบันทึกและส่งโค้ดผ่านคำสั่ง `git commit` และ `git push` ไปยัง Remote Repository บน GitHub ก่อน
> 2. **บราวเซอร์ของลูกค้าทำการแคชไฟล์ (Browser Cache)**: ไฟล์หน้าเว็บอาจถูกเก็บไว้ในหน่วยความจำชั่วคราวของเบราว์เซอร์เพื่อโหลดเร็วขึ้น สามารถแก้ไขได้โดยการกด `Ctrl + F5` หรือเปิดหน้าต่างไม่ระบุตัวตน (Incognito Mode) เพื่อบังคับดึงข้อมูลใหม่จากเซิร์ฟเวอร์

## 🚀 ขั้นตอนการนำไปใช้และอัปโหลดขึ้น GitHub Pages
สำหรับโปรเจกต์นี้ แนะนำให้ทำการคอมมิตอย่างน้อย 3 ครั้ง เพื่อแสดงพัฒนาการของงานตามกติกา ดังนี้:

### 💾 ขั้นตอนการทำ Git Commit (ทีละ Step):
1. **Commit 1: เริ่มต้นโปรเจกต์และ HTML**
   ```bash
   git init
   git add index.html
   git commit -m "Initialize project and create HTML structural layout"
   ```
2. **Commit 2: การตกแต่งด้วย CSS**
   ```bash
   git add style.css
   git commit -m "Design premium glassmorphism dark-mode styles in CSS"
   ```
3. **Commit 3: พฤติกรรมและตรรกะด้วย JS**
   ```bash
   git add script.js README.md
   git commit -m "Implement reactive GPA calculation logic and local storage in JavaScript"
   ```

### 🌍 ขั้นตอนการ Deploy ขึ้น GitHub Pages:
1. สร้าง **Public Repository** ใหม่บน GitHub โดยตั้งชื่อโปรเจกต์ที่สอดคล้องกัน
2. ผูก Repository ท้องถิ่นเข้ากับ GitHub:
   ```bash
   git remote add origin https://github.com/[ชื่อผู้ใช้ของคุณ]/[ชื่อโปรเจกต์].git
   git branch -M main
   git push -u origin main
   ```
3. เข้าไปที่หน้าเว็บ Repository บน GitHub -> ไปที่แถบ **Settings** (ขวาบน)
4. เลือกแถบเมนู **Pages** (แถบด้านซ้าย)
5. ในหัวข้อ **Build and deployment** -> ส่วนของ **Branch** ให้เปลี่ยนจาก `None` เป็น `main` และกดปุ่ม **Save**
6. รอประมาณ 1-2 นาที คุณจะได้ลิงก์หน้าเว็บสีเขียวปรากฏขึ้น สามารถเปิดใช้และนำลิงก์มาส่งได้ทันที!
